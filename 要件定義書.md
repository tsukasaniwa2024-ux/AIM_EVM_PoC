# 要件定義書

**プロジェクト名**：EVM 輸入帳票自動化システム  
**作成日**：2026-06-24  
**バージョン**：1.0  

---

## 1. 背景・目的

### 1.1 背景

AIM株式会社では中国から日本への輸入業務において、経理処理に使用する帳票（PDF・画像）から必要なデータを手作業で転記・計算している。この作業は工数がかかり、転記ミスのリスクも存在する。

### 1.2 目的

- 帳票データの読み取りをOCRで自動化し、手作業による転記を廃止する
- 計算処理を自動化して経理担当者の工数を削減する
- 処理結果をCSV / Excelで出力し、既存の経理フローに接続できるようにする
- データをDBに蓄積して将来的な分析・監査対応に備える

---

## 2. スコープ

### 2.1 対象

- 中国→日本 輸入に関する経理帳票（PDF形式 および 画像形式）
- 処理結果の出力（CSV / Excel）
- 処理履歴のDB保存

### 2.2 対象外

- 日本→中国 輸出帳票
- 会計システムへの直接連携（将来拡張）
- ユーザー認証・権限管理（将来拡張）

## 3. 利用者

| ロール | 説明 |
|--------|------|
| 経理担当者 | 帳票をアップロードし、結果を確認・出力する主な利用者 |

## 4. 機能要件

### 4.1 ファイルアップロード

| ID | 要件 |
|----|------|
| F-01 | ユーザーはブラウザ上でPDFファイルを1件選択できる |
| F-02 | ユーザーはブラウザ上で画像ファイル（PNG / JPG）を1件選択できる |
| F-03 | PDFと画像の両方が選択されていない場合、処理を実行できない |
| F-04 | ファイルサイズ上限は1ファイルあたり20MBとする |

### 4.2 OCRデータ抽出

| ID | 要件 |
|----|------|
| F-05 | PDFファイルからOCR APIを用いてテキストを抽出できる |
| F-06 | 画像ファイルからOCR APIを用いてテキストを抽出できる |
| F-07 | 抽出対象項目は「品名」「数量」「単価（人民元）」「合計金額（人民元）」「為替レート」「送料」「関税率」とする |
| F-08 | 抽出できなかった項目はNULLとして扱い、画面上に警告を表示する |
| F-09 | OCRプロバイダは抽象化レイヤーを介して差し替え可能とする |

### 4.3 計算処理

| ID | 要件 |
|----|------|
| F-10 | 抽出した人民元金額に為替レートを掛けて日本円換算額を算出する |
| F-11 | 日本円換算額に送料・関税を加算して輸入総コストを算出する |
| F-12 | 計算式・計算結果はすべてログとして保持する |

### 4.4 結果プレビュー

| ID | 要件 |
|----|------|
| F-13 | 「実行」ボタン押下後、計算結果を画面上の表形式で表示する |
| F-14 | プレビューにはOCR抽出値・計算中間値・最終結果をすべて含める |
| F-15 | 抽出値の手動修正フォームを表示する（将来拡張として設計に盛り込む） |

### 4.5 ファイル出力

| ID | 要件 |
|----|------|
| F-16 | 「CSVで出力」ボタン押下でCSVファイルをダウンロードできる |
| F-17 | 「Excelで出力」ボタン押下でExcel（.xlsx）ファイルをダウンロードできる |
| F-18 | 出力ファイルのファイル名には処理日時を含める（例：`import_20260624_153000.csv`） |

### 4.6 DB保存

| ID | 要件 |
|----|------|
| F-19 | 処理ごとに抽出データ・計算結果・処理日時をDBに保存する |
| F-20 | 開発環境ではSQLiteを使用する |
| F-21 | 本番環境（AIMイントラ）ではPostgreSQLへの切り替えを想定し、接続先をDATABASE_URLで切り替え可能とする |

## 5. 非機能要件

### 5.1 性能

| ID | 要件 |
|----|------|
| NF-01 | 1件の処理（OCR＋計算＋DB保存）を30秒以内に完了する |
| NF-02 | 同時利用者は1〜3名を想定する（ローカル環境） |

### 5.2 可用性・保守性

| ID | 要件 |
|----|------|
| NF-03 | Dockerで環境を再現できること（`docker-compose up --build`で起動） |
| NF-04 | OCRプロバイダを1ファイルの変更と`.env`設定のみで差し替えできること |
| NF-05 | 環境依存の設定値はすべて`.env`で管理すること（APIキー、DB URL等） |

### 5.3 セキュリティ

| ID | 要件 |
|----|------|
| NF-06 | APIキーをソースコードにハードコードしない |
| NF-07 | アップロードファイルはPDF / PNG / JPGのみ受け付けるバリデーションを行う |

### 5.4 拡張性

| ID | 要件 |
|----|------|
| NF-08 | 計算ロジックを`calculator.py`に集約し、ロジック変更がバックエンド全体に影響しない設計とする |
| NF-09 | DBのテーブル設計はカラム追加が容易なように正規化して設計する |

## 7. 用語定義

| 用語 | 説明 |
|------|------|
| 帳票 | 輸入業務で使用するインボイス・パッキングリスト等の書類 |
| OCR | 光学文字認識。画像・PDFからテキストを抽出する技術 |
| 為替レート | 人民元→日本円の換算レート。帳票上の値またはユーザー入力値を使用 |
| 輸入総コスト | 商品代金（円換算）＋送料＋関税 の合計 |
