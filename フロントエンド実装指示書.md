# フロントエンド実装指示書

## ファイル構成

```
frontend/
├── index.html
├── style.css
└── app.js
```

---

## index.html

以下のUI要素を含むHTMLを作成する。

- PDFファイル選択ボタン（`accept=".pdf"`）
- 画像ファイル選択ボタン（`accept=".jpg,.jpeg,.png"`）
- 「OCR解析・計算実行」ボタン（id: `run-btn`）
- ローディング表示エリア（id: `loading`、初期は非表示）
- 警告メッセージ表示エリア（id: `warnings`、初期は非表示）
- 計算結果プレビューテーブル（id: `result-table`、初期は非表示）
  - カラム：項目名 / 値 / 抽出元
- 「CSVで出力」ボタン（id: `csv-btn`、初期はdisabled）
- 「Excelで出力」ボタン（id: `excel-btn`、初期はdisabled）
- `<script src="app.js"></script>` を末尾に読み込む
- `<link rel="stylesheet" href="style.css">` を読み込む

---

## style.css

最低限見やすければOK。以下だけ対応する。

- 全体に適度な余白・中央寄せ
- テーブルにborderをつけて読みやすく
- ローディング中はスピナーまたは「処理中...」テキスト表示
- 警告メッセージは黄色背景で目立たせる
- ボタンのdisabled状態はグレーアウト

## app.js

### 1. ファイル選択バリデーション

```
PDFと画像、両方が選択されたときだけ run-btn を有効化する
片方でも未選択なら run-btn は disabled のまま
```

### 2. 「OCR解析・計算実行」ボタン押下時

```
① loading を表示、run-btn を disabled にする
② FormData に pdf_file と image_file をセットして
   POST /api/process へ送信（fetch）
③ レスポンス（JSON）を受け取る
④ loading を非表示にする
⑤ warnings があれば warnings エリアに表示
⑥ fields と calculations を result-table に展開する
   （後述のレスポンス形式を参照）
⑦ csv-btn と excel-btn を有効化する
⑧ エラー時はアラートまたはエラーエリアに表示する
```

### 3. レスポンスのJSON形式（参考）

```json
{
  "record_id": 1,
  "status": "ok",
  "warnings": ["unit_price_cny が抽出できませんでした"],
  "fields": [
    { "key": "item_name",     "value": "○○部品", "source": "pdf"   },
    { "key": "quantity",      "value": 100,       "source": "image" },
    { "key": "exchange_rate", "value": 20.5,      "source": "pdf"   }
  ],
  "calculations": [
    { "key": "total_jpy",        "value": 102500, "formula": "total_cny * exchange_rate" },
    { "key": "import_total_jpy", "value": 110625, "formula": "total_jpy + shipping_jpy + tariff_jpy" }
  ]
}
```

テーブルの表示イメージ：

| 項目名 | 値 | 抽出元 |
|--------|----|--------|
| item_name | ○○部品 | pdf |
| quantity | 100 | image |
| total_jpy | 102500 | calculated |
| import_total_jpy | 110625 | calculated |

### 4. 「CSVで出力」ボタン押下時

```
GET /api/export/csv/{record_id} をfetchし
Blob経由でファイルダウンロードさせる
```

### 5. 「Excelで出力」ボタン押下時

```
GET /api/export/excel/{record_id} をfetchし
Blob経由でファイルダウンロードさせる
```

### Blob経由ダウンロードの実装パターン

```javascript
const res = await fetch(`/api/export/csv/${recordId}`);
const blob = await res.blob();
const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url;
a.download = 'export.csv'; // ファイル名はレスポンスヘッダから取得してもOK
a.click();
URL.revokeObjectURL(url);
```

## 補足

- バックエンドは `http://localhost:8001` で起動している
- APIのベースURLは `app.js` の先頭で定数として定義しておくと変更しやすい
  ```javascript
  const API_BASE = 'http://localhost:8001';
  ```
- fetchのエラーハンドリング（network error / status !== 200）は必ず入れる
- `record_id` は `/api/process` のレスポンスから取得して変数に保持しておく
