## 3. 拡張性設計方針

### 3.1 基本方針

フィールドの型・種類・数はあらかじめ決めない。将来的に帳票フォーマットが変わっても**コードを触らずに対応できる**ことを優先する。

| 関心事 | 対応方針 |
|--------|----------|
| 抽出フィールドの追加・変更 | `extraction.yml` を編集するだけで対応 |
| 計算ロジックの変更 | `calculator.py` に集約、将来的に設定外出しも可能 |
| OCRプロバイダの差し替え | `OcrProvider` 基底クラスを継承して実装・`.env`で切り替え |
| DBカラムの追加 | EAV構造（key-value）のため、テーブル変更不要 |
| 重複データの扱い | マージ戦略を設定で定義（後述） |

### 3.2 抽出フィールド定義ファイル（extraction.yml）

フィールドの定義をコードから分離し、YAMLで管理する。

```yaml
# backend/config/extraction.yml
fields:
  - key: item_name
    label: ["品名", "商品名", "DESCRIPTION", "ITEM"]
    type: string
    source: any        # pdf | image | any（両方から探す）
    required: false

  - key: quantity
    label: ["数量", "QTY", "QUANTITY"]
    type: float
    source: any
    required: false

  - key: unit_price_cny
    label: ["単価", "UNIT PRICE", "单价"]
    type: float
    source: any
    required: false

  - key: exchange_rate
    label: ["レート", "RATE", "為替"]
    type: float
    source: any
    required: false

# フィールドは自由に追加・削除可能
```

- `source: any` にすると、PDFと画像の両方から検索する
- 同じキーが両ファイルから抽出された場合は **マージ戦略** に従う（3.3参照）
- `type` は `string` / `float` / `int` / `date` を想定（将来拡張）

### 3.3 重複データのマージ戦略

2ファイルから同じキーが取れた場合のルール：

| 戦略 | 説明 | デフォルト |
|------|------|-----------|
| `pdf_first` | PDFの値を優先、なければ画像の値を使う | ✅ |
| `image_first` | 画像の値を優先、なければPDFの値を使う | |
| `both` | 両方の値を別フィールドとして保持（`key_pdf` / `key_image`） | |

`.env` または `extraction.yml` のトップレベルで設定：

```yaml
merge_strategy: pdf_first   # pdf_first | image_first | both
```

## 6. モジュール設計

### 6.1 OCR抽象化レイヤー

```python
# backend/services/ocr/base.py
from abc import ABC, abstractmethod

class OcrProvider(ABC):
    @abstractmethod
    def extract_text(self, file_bytes: bytes, mime_type: str) -> str:
        """ファイルバイト列からテキストを抽出して返す"""
        ...

def get_ocr_provider() -> OcrProvider:
    provider = os.getenv("OCR_PROVIDER", "mock")
    if provider == "mock":
        from .mock import MockOcr
        return MockOcr()
    elif provider == "google_vision":
        from .google_vision import GoogleVisionOcr
        return GoogleVisionOcr()
    elif provider == "aws_textract":
        from .aws_textract import AwsTextractOcr
        return AwsTextractOcr()
    raise ValueError(f"Unknown OCR provider: {provider}")
```

### 6.2 動的データ抽出（extractor.py）

`extraction.yml` を読み込み、定義されたラベルで生テキストを検索する。フィールドはハードコードしない。

```python
def extract(raw_text: str, source: str, fields_config: list) -> dict:
    results = {}
    for field in fields_config:
        if field["source"] not in (source, "any"):
            continue
        for label in field["label"]:
            # ラベルの後ろの値を正規表現で抽出
            match = re.search(rf"{label}[^\d]*?([\d,.]+)", raw_text)
            if match:
                results[field["key"]] = {
                    "value": cast(match.group(1), field["type"]),
                    "source": source
                }
                break
    return results
```

### 6.3 計算ロジック（calculator.py）

計算式はこのファイルに集約する。将来的には計算式もYAML外出しを検討。

```python
def calculate(fields: dict) -> list:
    results = []
    # 各計算はfieldが存在する場合のみ実行（Noneスキップ）
    if fields.get("total_cny") and fields.get("exchange_rate"):
        total_jpy = fields["total_cny"] * fields["exchange_rate"]
        results.append({"key": "total_jpy", "value": total_jpy,
                         "formula": "total_cny * exchange_rate"})
    # ... 以降の計算式も同様
    return results
```

## 7. DB設計

### 7.1 設計方針

フィールドの種類・数が変動するため、EAV（Entity-Attribute-Value）構造を採用する。テーブル変更なしで任意のフィールドを追加・削除できる。

### 7.2 テーブル：`import_sessions`（処理セッション）

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| id | INTEGER | NOT NULL | 主キー（AUTO INCREMENT） |
| created_at | DATETIME | NOT NULL | 処理日時（UTC） |
| pdf_filename | VARCHAR(255) | NULL | PDFファイル名 |
| image_filename | VARCHAR(255) | NULL | 画像ファイル名 |
| merge_strategy | VARCHAR(50) | NULL | 使用したマージ戦略 |
| warnings | TEXT | NULL | 警告メッセージ（JSON文字列） |

### 7.3 テーブル：`import_fields`（抽出・計算フィールド）

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|
| id | INTEGER | NOT NULL | 主キー |
| session_id | INTEGER | NOT NULL | `import_sessions.id` への外部キー |
| key | VARCHAR(255) | NOT NULL | フィールド名（例: `item_name`） |
| value | TEXT | NULL | 値（文字列として保存） |
| value_type | VARCHAR(50) | NULL | 型情報（string / float / int / date） |
| source | VARCHAR(50) | NULL | 抽出元（pdf / image / calculated） |
| formula | TEXT | NULL | 計算フィールドの場合の計算式 |

### 7.4 接続設定

```python
DATABASE_URL = os.getenv("DATABASE_URL", "sqlite:///./evm_import.db")
engine = create_engine(DATABASE_URL)
```

`.env`の`DATABASE_URL`を変更するだけでSQLite↔PostgreSQLを切り替え可能。

## 9. エラーハンドリング方針

| エラー種別 | 対応 |
|-----------|------|
| ファイル未選択 | フロント側でバリデーション、ボタンをdisabledに |
| ファイル形式不正 | バックエンドで422を返し、画面にエラーメッセージ表示 |
| OCR API失敗 | 500を返し、エラー内容をログ出力・画面に表示 |
| フィールド抽出失敗 | NULL扱い＋warningsに追加、処理は続行 |
| DB保存失敗 | ログ出力、出力には影響させない（ベストエフォート） |
