## 4. API設計

### 4.1 エンドポイント一覧

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/` | フロントエンドHTML配信 |
| POST | `/api/process` | ファイルアップロード・OCR・計算・DB保存 |
| GET | `/api/export/csv/{record_id}` | CSV出力 |
| GET | `/api/export/excel/{record_id}` | Excel出力 |

### 4.2 POST `/api/process`

**リクエスト（multipart/form-data）**

| フィールド | 型 | 説明 |
|-----------|-----|------|
| pdf_file | File | PDF帳票（必須） |
| image_file | File | 画像帳票（必須） |

**レスポンス（JSON）**

```json
{
  "record_id": 1,
  "status": "ok",
  "warnings": ["為替レートが抽出できませんでした"],
  "result": {
    "item_name": "○○部品",
    "quantity": 100,
    "unit_price_cny": 50.00,
    "total_cny": 5000.00,
    "exchange_rate": 20.5,
    "total_jpy": 102500,
    "shipping_jpy": 3000,
    "tariff_rate": 0.05,
    "tariff_jpy": 5125,
    "import_total_jpy": 110625
  }
}
```

**エラーレスポンス**

```json
{
  "status": "error",
  "message": "PDFファイルが指定されていません"
}
```

### 4.3 GET `/api/export/csv/{record_id}`

- Content-Type: `text/csv`
- ファイル名: `import_{YYYYMMDD}_{HHMMSS}.csv`

### 4.4 GET `/api/export/excel/{record_id}`

- Content-Type: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- ファイル名: `import_{YYYYMMDD}_{HHMMSS}.xlsx`

## 5. モジュール設計

### 5.1 OCR抽象化レイヤー

```python
# backend/services/ocr/base.py
from abc import ABC, abstractmethod

class OcrProvider(ABC):
    @abstractmethod
    def extract_text(self, file_bytes: bytes, mime_type: str) -> str:
        """ファイルバイト列からテキストを抽出して返す"""
        ...

def get_ocr_provider() -> OcrProvider:
    """環境変数 OCR_PROVIDER に応じてプロバイダを返すファクトリ関数"""
    provider = os.getenv("OCR_PROVIDER", "google_vision")
    if provider == "google_vision":
        from .google_vision import GoogleVisionOcr
        return GoogleVisionOcr()
    elif provider == "aws_textract":
        from .aws_textract import AwsTextractOcr
        return AwsTextractOcr()
    raise ValueError(f"Unknown OCR provider: {provider}")
```

### 5.2 データ抽出（extractor.py）

OCRで取得した生テキストから正規表現を用いて各フィールドを抽出する。

| フィールド | 抽出元 | 抽出方法 |
|-----------|--------|----------|
| 品名 | PDF | 正規表現（"商品名"、"品名"等のラベル後の値） |
| 数量 | PDF | 正規表現（"数量"、"QTY"等のラベル後の数値） |
| 単価（元） | PDF | 正規表現（"単価"、"UNIT PRICE"等のラベル後の数値） |
| 合計金額（元） | PDF | 正規表現（"合計"、"TOTAL"等のラベル後の数値） |
| 為替レート | 画像 | 正規表現（"レート"、"RATE"等のラベル後の数値） |
| 送料 | 画像 | 正規表現（"送料"、"FREIGHT"等のラベル後の数値） |
| 関税率 | 画像 | 正規表現（"関税"、"TARIFF"等のラベル後の数値） |

抽出に失敗したフィールドはNullとしてwarningsリストに追加する。

### 5.3 計算ロジック（calculator.py）

```python
# 計算式定義
total_jpy       = total_cny × exchange_rate
tariff_jpy      = total_jpy × tariff_rate
import_total    = total_jpy + shipping_jpy + tariff_jpy
```

計算式はこのファイルに集約し、他モジュールから計算ロジックを参照しない。

### 5.4 出力（exporter.py）

- CSV：`csv`標準ライブラリ + `io.StringIO`でバイト列化
- Excel：`openpyxl`でワークブック生成、ヘッダー行に背景色を付与

## 8. エラーハンドリング方針

| エラー種別 | 対応 |
|-----------|------|
| ファイル未選択 | フロント側でバリデーション、ボタンをdisabledに |
| ファイル形式不正 | バックエンドで422を返し、画面にエラーメッセージ表示 |
| OCR API失敗 | 500を返し、エラー内容をログ出力・画面に表示 |
| フィールド抽出失敗 | NULL扱い＋warningsに追加、処理は続行 |
| DB保存失敗 | ログ出力、出力には影響させない（ベストエフォート） |

## 9. 開発・テスト方針

### 9.1 OCRモック

APIキー確定前の開発期間中は、`OcrProvider`を継承したモッククラスを使用する。

```python
class MockOcr(OcrProvider):
    def extract_text(self, file_bytes, mime_type):
        return "品名: テスト部品\n数量: 100\n単価: 50.00\n合計: 5000.00\nレート: 20.5\n送料: 3000\n関税率: 5%"
```

`.env`で`OCR_PROVIDER=mock`を指定することで切り替える。
