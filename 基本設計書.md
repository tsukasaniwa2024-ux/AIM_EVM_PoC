## 3. 拡張性設計方針

| 関心事 | 対応方針 |
|--------|----------|
| 抽出フィールドの追加・変更 | OpenAIへのプロンプトを修正するだけで対応 |
| 計算ロジックの変更 | `calculator.py` に集約 |
| OCRプロバイダの差し替え | `OcrProvider` 基底クラスを継承・`.env`の`OCR_PROVIDER`で切り替え |
| DBカラムの追加 | EAV構造のため、テーブル変更不要 |
| 品目明細カラムの変更 | フロントが動的生成のため、コード変更不要 |

## 5. API設計

### 5.1 エンドポイント一覧

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/` | フロントエンドHTML配信 |
| GET | `/health` | ヘルスチェック |
| POST | `/api/process` | ファイルアップロード・OCR・計算・DB保存 |
| GET | `/api/export/csv/{record_id}` | CSV出力 |
| GET | `/api/export/excel/{record_id}` | Excel出力 |

### 5.2 POST `/api/process`

**リクエスト（multipart/form-data）**

| フィールド | 型 | 説明 |
|-----------|-----|------|
| pdf_file | File | PDF帳票（必須） |
| image_file | File | 画像帳票（必須） |
| exchange_rate | float | 為替レート（手動入力時のみ、1〜999） |

**レスポンス（JSON）**

```json
{
  "status": "ok",
  "record_id": 1,
  "pdf": "COMMERCIAL_INVOICE.pdf",
  "image": "PACKING_List.jpg",
  "basic_info": [
    {"key": "invoice_no", "value": "IV-250414", "source": "pdf", "value_type": "string"},
    {"key": "exchange_rate", "value": 22.0, "source": "manual", "value_type": "float"}
  ],
  "items": [
    {
      "item_no": "A001",
      "description": "Laptop",
      "quantity": 5,
      "amount": 500000.0,
      "amount(円換算)": "11,000,000円"
    }
  ],
  "warnings": []
}
```

**エラーレスポンス**

```json
{"status": "error", "message": "為替レートは1〜999の範囲で入力してください。"}
```

### 5.3 GET `/api/export/csv/{record_id}` / `/api/export/excel/{record_id}`

- ファイル名: `import_{YYYYMMDD}_{HHMMSS}.csv` / `.xlsx`
- CSV：`【基礎情報】`と`【品目明細】`の2セクション構成
- Excel：`基礎情報`シートと`品目明細`シートの2シート構成

## 6. モジュール設計

### 6.1 OCR抽象化レイヤー（ocr/base.py）

```python
class OcrProvider(ABC):
    @abstractmethod
    def extract(self, file_bytes: bytes, mime_type: str) -> dict:
        # 戻り値: {"basic_info": {...}, "items": [...]}
        ...

def get_ocr_provider() -> OcrProvider:
    provider = os.getenv("OCR_PROVIDER", "mock")
    # mock / openai を切り替え
```

### 6.2 OpenAI Vision実装（ocr/openai_vision.py）

- PDFは`pdf2image`で先頭ページを画像変換してから送信
- プロンプトで`basic_info`と`items`の2構造でJSON返却を指示
- `basic_info`：日付・番号・取引先・合計等のヘッダー情報
- `items`：品番・品名・数量・金額等の明細行

### 6.3 マージ処理（extractor.py）

- `basic_info`はPDF・画像の重複キーを`pdf_first`戦略でマージ
- `items`はPDF優先、なければ画像を使用
- nullフィールドは除外

### 6.4 計算ロジック（calculator.py）

```python
# 品目ごとの円換算
amount(円換算) = amount × exchange_rate
```

### 6.5 出力（exporter.py）

| 形式 | 構成 |
|------|------|
| CSV | 【基礎情報】セクション + 【品目明細】セクション（BOM付きUTF-8） |
| Excel | 「基礎情報」シート + 「品目明細」シート（openpyxl） |

## 7. DB設計

### 7.1 テーブル：`import_sessions`

| カラム名 | 型 | NULL | 説明 |
|---------|-----|------|------|

## 9. エラーハンドリング方針

| エラー種別 | 対応 |
|-----------|------|
| ファイル未選択 | フロント側でバリデーション、実行ボタンをdisabledに |
| 為替レート範囲外 | バックエンドでエラーレスポンス、画面にメッセージ表示 |
| 為替レート未入力 | バックエンドでエラーレスポンス、画面にメッセージ表示 |
| OCR API失敗 | 500を返し、画面にエラーメッセージ表示 |
| フィールド抽出失敗 | 「抽出できず」として表示、処理は続行 |
| DB保存失敗 | ログ出力、出力には影響させない（ベストエフォート） |
